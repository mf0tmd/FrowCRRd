# Руководство пользователя FrowCRRd

Актуально для текущей версии программы (GUI + native core).

## 1. Что это за программа

`FrowCRRd` — настольный симулятор 2D-полёта ракеты:

- сборка конфигурации ракеты в GUI;
- расчёт траектории в нативном C++ ядре;
- просмотр телеметрии, событий и графиков воспроизведения.

Важно: это **2D point-mass модель**, не 6DOF.

## 2. Что нужно установить

### Обязательно

- Node.js 20+
- npm
- CMake 3.20+
- MSYS2 UCRT64 (для сборки нативного runner на Windows)

См. подробности в:

- `README.md` (корень проекта)
- `Gui/README.md`

## 3. Быстрый запуск программы

Из папки `Gui`:

```powershell
npm install
npm run dev:desktop:core
```

Если PowerShell блокирует `npm`:

```powershell
npm.cmd install
npm.cmd run dev:desktop:core
```

## 4. Основной рабочий поток

### Шаг 1: Main Menu

- `Run` → переход в `SetupView`
- `Examples` → загрузка готовых конфигураций
- `Documentation` / `Tutorial` / `Settings`

### Шаг 2: SetupView (быстрая сборка)

Здесь задаются базовые параметры:

- название миссии;
- количество ступеней;
- для каждой ступени: структурная масса, диаметр, масса топлива, число двигателей.

Для **последней ступени** в SetupView есть быстрый блок восстановления:

- число парашютов;
- тип каждого парашюта (`Main` / `Drogue`).

После этого нажмите `Confirm Project`.

### Шаг 3: EditorView (детальная настройка)

Режим `Builder`:

- дерево ракеты (ступени, баки, двигатели, pitch program, parachutes);
- точная настройка параметров;
- графики профиля тяги и pitch-программы.

Режим `Simulation`:

- запуск симуляции;
- главный график траектории;
- мини-графики телеметрии;
- журнал событий.

## 5. Важные текущие правила

### 5.1 Двигатели

- В UI нельзя переключать тип двигателя (режим фиксированного типа в текущем UX).
- Настраиваются: число двигателей, тяга, расход массы, масса двигателя, профиль throttle.

### 5.2 Pitch Program

- Поддерживается диапазон угла **0..360** градусов.
- Если pitch включён и на старте угол сильно не вертикальный, старт может быть невозможен.

### 5.3 Аэродинамика

- Аэродинамическое сопротивление всегда включено.
- Отдельного переключателя drag нет.

### 5.4 Парашюты

- Поддерживаются несколько парашютов.
- Парашюты настраиваются в дереве `Parachutes` в Builder.
- Ограничение ядра: **все парашюты в одной конфигурации должны использовать один и тот же режим** триггера (`time` / `altitude` / `speed`).

### 5.5 Остановка симуляции

В настройках симуляции есть `Simulate until ground impact`.

- Если выключено: симуляция идёт до `tMax`.
- Если включено: runner может автоматически расширять горизонт, пока не найдёт удар о землю (внутренний предел по времени есть).

## 6. События и графики

В журнале и на траектории отображаются события, включая:

- ignition / liftoff;
- изменения throttle/pitch;
- separation/fairing;
- apogee;
- impact;
- **engine burnout** (выгорание двигателей).

События синхронизированы с ползунком playback.

## 7. Почему симуляция может быть «бредовой»

Частые причины:

- начальная вертикальная тяга меньше веса ракеты;
- слишком агрессивный pitch на старте;
- нулевой/некорректный расход массы у двигателя;
- конфликтные настройки парашютов;
- некорректные массы ступеней.

Программа показывает ошибку/предупреждение, если старт невозможен (по условию тяга vs вес).

## 8. Примеры конфигураций

В `Examples` есть 3 пресета:

- `Suborbital`;
- `Two-stage`;
- `Recovery`.

Они актуализированы под текущий bridge/core и используются как стартовые шаблоны.

## 9. Экспорт

В Simulation доступны экспорты:

## 10. Мини-чеклист перед запуском

- заполнено название миссии;
- у ступеней с топливом есть двигатели;
- у двигателей положительные `thrust` и `massFlow`;
- pitch на старте адекватен;
- парашюты (если используются) имеют корректные диапазоны и общий режим.
